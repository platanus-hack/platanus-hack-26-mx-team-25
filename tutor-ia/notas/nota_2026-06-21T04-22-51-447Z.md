# Diagrama de Flujo: Petición HTTPS

## Descripción
Una petición HTTPS sigue estos pasos:

1. **DNS Lookup**: El navegador traduce el dominio a dirección IP
2. **Conexión TCP**: Se establece conexión al servidor (puerto 443)
3. **TLS Handshake**: Se negocia el cifrado seguro
4. **HTTPS Request**: El cliente envía la solicitud cifrada
5. **Procesamiento**: El servidor procesa la petición
6. **HTTPS Response**: El servidor devuelve la respuesta cifrada
7. **Renderizado**: El navegador muestra el contenido
